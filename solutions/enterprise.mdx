## Team Management

### Roles

جستجوپذیر Enterprise ships with four built-in roles, each scoped to the level of access appropriate for that function:

| Role | What they can do |
|---|---|
| **Admin** | Full platform access: manage workspaces, create/delete users, configure SSO, manage billing, and view all audit logs. |
| **Manager** | Create and configure workspaces and keyword tracking; generate and share reports; manage Analyst and Viewer access within their assigned workspaces. |
| **Analyst** | View all data within assigned workspaces; run audits; export reports; configure AI Agent alerts. Cannot manage users or billing. |
| **Viewer** | Read-only access to dashboards and reports within assigned workspaces. Cannot export data or modify any settings. |

Custom roles with granular permission overrides are available on request — contact your account manager to discuss your organisation's access model.

### API Key Management

Create and scope API keys per integration use case from **Settings → API Keys**. Each key can be restricted to specific workspaces, limited to read-only or read-write operations, and given an expiry date. Rotate or revoke keys instantly without affecting other integrations.

### Organisation Workspace Structure

Enterprise accounts support a hierarchical workspace structure: an organisation-level parent workspace containing multiple child workspaces scoped to business units, product lines, regional markets, or individual brands. Admins see the full hierarchy; team members see only the workspaces assigned to their role.
